The core contract all model providers implement.

**File:** `src/phase1/orchestrator.ts:12-15`

```typescript
export interface LLMProvider {
  call(prompt: string): Promise<string>;
  callWithTools?(messages: Message[], tools: ToolSchema[]): Promise<LLMResponse>;

```

Implementations in `src/providers/`:
- `ClaudeProvider` — implements `call` + `callWithTools` (Anthropic tool_use format)

## 2. Orchestrator Retry Pattern (Ralph Wiggum)

Kill-and-restart: each attempt gets a fresh agent with only the previous test output as context.

**File:** `src/phase1/orchestrator.ts:94-122`

```typescript
for (this.attempt = 1; this.attempt <= this.config.maxRetries; this.attempt++) {
  const log = await this.executeSingleAttempt(
    task, this.attempt,
    this.attempt > 1 ? this.lastTestOutput : undefined
  );
  logs.push(log);
  if (log.status === 'success') break;
  // Failure/restart — Ralph Wiggum: kill this agent, restart fresh
}
```

Key insight: only `lastTestOutput` carries over between attempts — no accumulated state.

## 5. Tool-Calling Loop (Agentic)

The inner execution loop — iteratively calls tools until the model returns text.

**File:** `src/phase1/orchestrator.ts:177-236`

```typescript
// 1. Build messages array from task prompt
// 2. Call provider.callWithTools(messages, tools)
// 3. If TextResponse -> done
// 4. If ToolCallResponse -> execute each tool via ToolShed (3PM gate first)
// 5. Append tool results to messages, loop back to step 2
// 6. Max iterations safety (default 10)
```

3PM approval gate fires before write/execute tool calls. `bash_execute` is classified as `critical` with mandatory approval.
